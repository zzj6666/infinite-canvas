# 快速开始

当前版本是**小团体自托管多用户**：需要服务端。管理员配置 API 后，普通用户登录即可使用，数据保存在服务器。

## Docker 一键启动（推荐）

```bash
git clone <your-fork-url>
cd infinite-canvas
cp .env.example .env
# 编辑 .env，至少修改 ADMIN_PASSWORD
docker compose up --build -d
```

访问：

```text
http://localhost:3000
```

默认管理员（可用环境变量覆盖）：

- 用户名：`admin`
- 密码：`admin123`（请务必修改）

数据目录挂载在 Docker volume `ic-data`（或本地 `./data`）。

## 本地开发

需要同时启动服务端与前端：

```bash
# 终端 1：服务端（默认 3010，避免和本机其他项目抢 3001）
cd server
bun install
PORT=3010 DATA_DIR=../data bun run dev

# 终端 2：前端（代理 /api 到 3010）
cd web
bun install
bun run dev
```

访问 `http://localhost:3000`，使用管理员账号登录。

## 首次使用建议

1. 使用管理员登录。
2. 打开「配置」→ 渠道，填写 Base URL、API Key 与模型，点「保存到服务器」。
3. 进入「用户管理」创建普通用户。
4. 普通用户登录后无需再配 Key，可直接使用画布生图等功能。

## 说明

- 画布、素材、提示词、媒体文件按用户隔离，保存在服务端。
- AI Key 只保存在服务端，浏览器不会拿到明文 Key。
- 不开放自助注册；账号由管理员创建。
