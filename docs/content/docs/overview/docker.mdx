# Docker 部署

如果你希望在自己的机器或服务器上运行项目，可以直接使用 Docker Compose。

## 使用发布镜像

```bash
git clone git@github.com:basketikun/infinite-canvas.git
cd infinite-canvas
docker compose up -d
```

启动后访问：

```text
http://localhost:3000
```

## 本地构建镜像

如果需要基于当前源码构建镜像：

```bash
docker compose -f docker-compose.local.yml up -d --build
```

## 反向代理与方舟参考素材

如通过 HTTPS 反向代理部署，且需要在火山方舟 Seedance 任务中使用本地图片、视频或音频参考，请在 `.env` 设置站点的公网地址：

```bash
PUBLIC_ORIGIN=https://canvas.example.com
```

方舟将通过该地址读取短时共享的参考素材。

## 文档站镜像

文档站位于 `docs/` 目录，按带服务端能力的 Next.js standalone 应用单独构建，不打进主应用镜像。

使用发布镜像：

```bash
cd docs
docker compose up -d
```

基于当前源码本地构建：

```bash
cd docs
docker compose -f docker-compose.local.yml up -d --build
```

## 数据说明

主应用镜像启动服务端 API 和前端静态资源。画布、我的素材和媒体文件保存在 `DATA_DIR`；Docker Compose 默认使用命名 volume 持久化，生产环境应定期备份该数据卷。
