# 待测试

- 多用户自托管：未登录跳转 `/login`；管理员可用环境变量首次启动创建；可创建/禁用/重置普通用户密码。
- 系统 AI 配置：管理员在配置页填写渠道 Key 并「保存到服务器」后，普通用户无需配 Key 即可在画布生图；浏览器 Network 仅出现 `/api/ai/*`。
- 数据隔离：用户 A 的画布、素材和媒体，用户 B 不可见；刷新或换浏览器同账号数据仍在。
- Docker：`docker compose up --build` 后单端口访问前端与 API，数据落在 volume。
- 移除 Codex / 本地 Agent：顶栏无 Agent/Codex 入口，画布顶栏无 Agent 按钮。
- 站点裁剪：网页端已移除「生图工作台」「视频创作台」导航与路由；提示词库改为服务端共享管理，不再拉取第三方源。需验证顶栏为画布 / 提示词库 / 素材，配置通过右上角图标打开，旧路径 `/image`、`/video` 进入 404。
- 共享提示词库：任一登录用户新建、编辑或删除提示词后，其他登录用户刷新可见相同结果；画布提示词选择弹窗只展示服务端共享提示词。
- 共享提示词升级：已有服务端数据目录启动后应保留历史提示词，并可正常新建、编辑和删除共享提示词。
- 个人资料：登录用户可在“配置与用户偏好”的“个人与偏好”页修改显示名；修改密码需校验当前密码和二次确认，成功后所有设备均需重新登录。
- 提示词库界面：标题区应明确共享属性；搜索、分类和标签在同一检索面板内可用；筛选后可清除条件；卡片在桌面和移动端应正常展示、复制、编辑和删除。
- 我的素材界面：标题区、导入导出和新建操作应清晰可用；搜索与类型筛选在同一检索面板内可用；卡片在桌面和移动端可正常查看、复制或下载、编辑和删除。
- 配置界面：渠道、模型、个人与偏好应有清晰分区；渠道卡片可展开编辑、启用和拉取模型；普通用户打开配置后应直接展开个人与偏好，不显示单独的 Tab。
- 火山方舟渠道：填写 Base URL 和 API Key 后可拉取模型；Agent Plan（`/api/plan/v3`）应加载当前套餐可用模型。
- 视频生成设置：Seedance 2.0 标准版可选择 480p、720p、1080p、4K；Fast 仅显示 480p、720p，Mini 最高显示 1080p；画面比例卡应紧凑显示比例值；设置弹层在浅色与深色画布主题下均可正常操作。
- 图片模型设置：GPT Image 2 显示生成质量、像素尺寸、自定义宽高和常用比例；`gpt-image-gemini-3-flash` 映射 Nano Banana 2，显示 512/1K/2K/4K 与其官方比例；`gpt-image-gemini-3-pro` 映射 Nano Banana Pro，仅显示 1K/2K/4K 与其官方比例；不同模型的标题、分区和按钮应保持统一，比例选项应同时显示画幅图形和比例值，切换模型后摘要和实际请求参数应同步变化。
- 火山方舟 Seedream：已拉取的 `doubao-seedream-5.0-lite` 应自动进入图片模型可选项；设置面板显示 2K/3K/4K 与官方比例；文生图和最多 14 张参考图的图生图均通过方舟图片生成接口提交，蒙版编辑应明确提示暂不支持。
- 模型与生成偏好界面：图像、视频、文本、语音模型应分别展示可选范围和默认模型；画布生成、音频输出和系统提示词应独立分组，原有配置读写保持正常。
- 画布节点名称：每个节点保留 `title` 名称字段，默认名称显示在节点上方，长名称应限制在节点宽度内单行省略且悬停可查看完整内容；双击名称可直接编辑，节点信息弹窗和 JSON 可查看该字段。
- 画布节点工具条：节点顶部工具条改为点击选中节点后显示，单纯鼠标移入 / 移出节点不应再频繁弹出。
- 上游画布修复：画布内设置浮层、下拉框和弹窗应可正常滚动；节点生成完成后提示词输入内容应继续保留；选择或平移画布时节点交互应保持流畅。
- 画布鼠标交互：在空白处按住鼠标中键拖动应平移画布；按住鼠标左键拖动应框选节点，`Shift + 左键拖动`应追加框选；点击空白处应关闭节点详情和顶部工具条，鼠标悬停未选中节点不应再次显示关联详情。
- 画布生图设置：设置弹层应按质量、尺寸、宽高比和生成张数分段展示；当前选项有明确高亮，宽高比和张数快捷选项紧凑排列，所有原有参数仍可修改。
- 画布生图尺寸：图像设置不显示“16 倍数对齐”开关；名称匹配 `image2`、`image-2` 的模型手动输入宽高应自动补齐为 16 的倍数，其他模型应保持原始输入尺寸。
- `gpt-image-2` 生图：常用比例预设应请求实际像素尺寸；自定义尺寸应在提交前校验为 16 的倍数、最长边小于 3840px、比例不超过 3:1、总像素在 655,360 至 8,294,400 之间；旧的比例配置如 `1:1` 应自动转换为有效尺寸。
- 图像设置：生成质量与画质分辨率应分开展示；切换 1K / 2K / 4K 时应保持当前宽高比并使用有效尺寸；生成张数只可选 1 至 4 张，不再提供自定义输入，工作台、画布和服务端均应限制为最多 4 张。
- 局部遮罩编辑：使用 `gpt-image-2` 时应以官方 `image[]` 字段上传参考图；涂抹区域被修改，未遮罩区域的构图、主体、光影、色彩、相机角度和细节应尽量保持不变。
- 多参考图提示词：从生成节点输入端或参考图输出端建立连接时，所有参考图都应成为生成输入；提示词输入不应闪烁或清空，可在任意文字后输入 `@` 并逐一插入多个图片引用。
- 多参考图生成：提示词应列出“图片编号（图片名称）”，并提示明确每张图的保留、迁移或组合用途；`gpt-image-2` 在前端和服务端均应拒绝超过 16 张参考图的请求。
- 生成提示词引用：插入 `@图片1` 等参考图标签后，输入框内应高亮显示该标签；连接参考图或选择文本时，普通提示词内容不应闪烁或消失。
- 生成提示词引用：光标紧邻已插入的图片引用标签时，按 Backspace 或 Delete 应一次删除整个标签及其后的空格。
- 生成提示词引用：图片引用应渲染为带独立内外边距的真实标签，不依赖空格补偿；标签间或标签与普通文字之间的间距、光标定位和整体删除应保持正确。
- 生成提示词引用：插入图片引用后使用中文输入法输入时，首个拼音字母不应残留在已确认的中文字符前。
- 图像设置与提示词编辑：工作台切换图片模型后应使用该模型的尺寸规则；自定义宽高比切换 1K / 2K / 4K 应保持接近原比例；`Shift + Enter` 换行和中文输入法替换选中文字均应正常保留。
- 模型展示：模型选择器和生成记录仅显示模型名称，不显示供应商渠道名称；选择后仍应请求所选渠道。
- 模型下拉框：画布内非全宽模型选择器应随模型名称收紧，下拉选项不应保留固定的大面积空白。
- 生图 4K：16:9 预设应为 `3840x2160`，9:16 预设应为 `2160x3840`，服务端应接受最长边为 3840px 的尺寸。
- 局部遮罩编辑：结果节点只显示用户输入的编辑要求，不显示内部的蒙版区域保护提示词；首次和重试请求均应仅修改蒙版区域。
- 画布光标：普通悬停不显示手型光标，按住鼠标中键平移期间才显示手型光标，松开后恢复元素原有光标。
- 画布菜单：打开左上角画布菜单后，点击画布空白处应收起菜单。
- 渠道模型拉取：新增或未保存的渠道二点击“拉取模型”时，应使用该渠道当前填写的 Base URL、API Key 和调用格式，不得返回渠道一的模型列表。
- AI 渠道开关：关闭渠道后保留其配置和模型列表，但不再出现在模型选择中且不能生成；重新开启后可再次选择和调用。
- 登录页：浅色/深色主题下应展示居中登录表单与画布背景；用户名、密码校验、登录成功后进入“我的画布”和失败提示保持正常。
- 图片节点工具条：在深色画布主题下，图片节点顶部的操作条、悬停项和提示气泡应使用深色主题颜色。
- 图片节点工具条：默认应以紧凑图标显示，悬停可查看名称；“更多”设置中可重新开启按钮文字。
- 节点信息：图片和视频节点应显示其实际媒体像素尺寸，不应误显示画布中的缩放尺寸。
- 火山方舟 Seedance 2.0：渠道格式可选择“火山方舟”并自动填入 `https://ark.cn-beijing.volces.com/api/v3`；模型 ID 需手动填写。正式模型应通过官方内容生成任务接口提交文本、图片、视频和音频参考；标准版可选 4K，Fast/Mini 不可选；本地参考素材应以短时只读链接供方舟读取。
- 火山方舟反向代理：通过 HTTPS 域名部署时，设置 `PUBLIC_ORIGIN` 为公网地址后，Seedance 应能读取本地图片、视频和音频参考素材。
- 全站主题：浅色/深色模式下，顶部导航、画布库、提示词库、素材库、配置、用户管理和 404 页面应采用一致的暖灰背景、边框、卡片与浮层层级；桌面和移动端导航、列表、表格均应保持可用。
- 渠道配置：多个渠道卡片可独立展开或收起；收起后仍可查看摘要、切换启用状态、拉取模型或删除，新增渠道应自动展开。
- 画布图片缩略图：新生成、上传、裁剪、切分和放大的图片节点应加载压缩缩略图；已浏览的缩略图应在当前浏览器缓存七天并优先从本地读取，过期后重新下载；点击“查看图片详情”时应加载并展示原图，旧节点仍能正常显示。
- 图片组主图：切换主图或删除当前主图后，根节点的缩略图、图片详情、下载和 AI 参考应一致使用新的主图；进入画布时应清理过期缩略图缓存。
- 画布组节点：组内子节点优先响应点击和拖拽；当鼠标位于组内节点上时，应移动 / 编辑该节点，而不是移动整个组。
- 画布组节点：工具栏新增“组”节点，普通节点拖入组区域会高亮目标组，松手后自动吸附进组；拖出组区域会解除归属，拖动组节点会一起移动组内子节点。
- GitHub Pages：新增版本 tag 触发的前端静态站点自动发布 workflow，需在仓库 Pages 设置中选择 GitHub Actions 并验证路由刷新回退。
- 顶部导航：缩小桌面顶部导航栏高度，导航项下划线和内容垂直对齐需确认。
- 配置与用户偏好：新增独立顶部导航页面，并在页面和弹窗中复用同一套配置面板。
- 配置与用户偏好：每个模型渠道新增 OpenAI / Gemini 调用格式选择，默认 OpenAI，切换默认格式时同步更新默认 Base URL；渠道 Tab 增加紧凑醒目的提醒，说明模型是否可选需要到“模型”Tab 选择。
- Gemini 调用格式：支持拉取 Gemini 模型列表，并用于文本问答、基础生图和图生图请求。
- Gemini 生图：画布生图会把尺寸配置转换为 Gemini `responseFormat.image.aspectRatio` 传入，需验证 1:1、9:16、16:9 及自定义像素尺寸的输出比例。
- 音频和视频生成：选择 Gemini 调用格式时先给出不支持提示，仍需使用 OpenAI 兼容渠道。
- 数据保存：配置页不应显示 WebDAV，同一账号在不同浏览器登录后应读取相同的服务端画布和素材；画布库不应显示 ZIP 导入或导出入口。
- 前端构建：移除 Next.js，改为 Vite + React Router，并将前端目录调整为 `pages` / `components` / `stores` / `lib` / `layouts` / `styles`，需验证首页、素材、提示词库和画布路由跳转。
- 前端构建：修复 TypeScript 严格检查下的构建报错。
- Vercel 部署：新增 `web` 和 `docs` 两个子项目的 Vercel 配置，需分别验证静态前端和文档站部署。
- Docker 部署：运行镜像从 Node serve 改为 nginx 静态托管，需验证前端路由刷新回退。
- 环境变量：移除过期的后端 `.env.example` 示例文件。
- 画布生成配置：修复切换文本/视频/音频模式时模型选择仍显示为生图模型（gpt-image-2）的问题，切换模式后应自动匹配对应能力的默认模型。
